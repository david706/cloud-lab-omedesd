Durante el curso construirás un único proyecto profesional. No entregarás una colección de prácticas aisladas: diseñarás, desplegarás y documentarás una infraestructura de virtualización, contenedores, cloud e inteligencia artificial.

1. ¿Qué vas a construir?

RA1 · Virtualización

Crearás la infraestructura virtual que actuará como base del proyecto.

RA3 · Contenedores

Desplegarás servicios reproducibles mediante Docker y una plataforma de IA.

RA2 · Cloud

Ampliarás o sustituirás partes de la solución utilizando servicios cloud.

Producto final: una arquitectura profesional documentada y justificada, no una memoria escrita al final del curso.

2. Tu repositorio personal

Cada alumno utilizará un repositorio propio con este formato:

cloud-lab-apellido-inicial

Ejemplo:

cloud-lab-garcia-j

Recibe la plantilla

Descarga y descomprime la plantilla común.

Personaliza

Cambia nombre, grupo, URL y datos del proyecto.

Crea GitHub

Crea un repositorio vacío en tu cuenta.

Conecta y sube

Añade la URL remota y realiza tu primer push.

3. Markdown: escribir sencillo, publicar bonito

Solo editarás archivos .md. MkDocs convertirá el contenido en una web ordenada y profesional.

Lo que escribes

# Despliegue de Docker

## Objetivo

Desplegar los servicios del proyecto.

## Evidencia

![Docker funcionando](imagenes/docker.png)

```bash
docker ps
```

Lo que consigues

Una página con títulos, navegación, imágenes, tablas, avisos, código resaltado y diseño adaptable.

No necesitas escribir HTML ni diseñar la web.

4. El estándar Cloud Lab

Todos los hitos siguen la misma estructura para facilitar el trabajo, la revisión y la evaluación.

SecciónQué debes incluir
ObjetivoQué vas a construir y qué necesidad resuelve.
Datos de conexiónEquipo, IP o URL, puerto, usuario y método de acceso.
ArquitecturaDiagrama y explicación de los componentes.
ProcedimientoPasos técnicos relevantes y decisiones.
EvidenciasCapturas, comandos, configuraciones y pruebas verificables.
IncidenciasProblema, diagnóstico y solución.
ReflexiónAprendizaje, mejoras y aplicación profesional.

5. Datos de conexión

En cada hito completarás una tabla parecida a esta:

DatoEjemplo
Equipo o servicioServidor Docker
IP o URL192.168.50.25
Puerto22 / 9443
Usuarioalumno
Método de accesoSSH o interfaz web
Nunca publiques: contraseñas, claves privadas, tokens, códigos de recuperación ni credenciales de AWS.

6. Git y los commits

Los commits registran la evolución real del proyecto. No se evaluará la cantidad, sino la coherencia del proceso.

git init
git add .
git commit -m "Inicio de Cloud Lab"
git branch -M main
git remote add origin URL_DE_TU_REPOSITORIO
git push -u origin main

Ejemplos de mensajes adecuados

  • Documento la red del laboratorio virtual
  • Añado pruebas del despliegue de Docker
  • Corrijo el diagrama de la arquitectura cloud

GitHub mostrará automáticamente la fecha, la hora, el autor y el historial de cambios.

8. Actividad de la Sesión 0

  1. Descarga y descomprime la plantilla.
  2. Renombra la carpeta con el formato indicado.
  3. Cambia tus datos en la portada.
  4. Crea tu repositorio vacío en GitHub.
  5. Conecta la carpeta con tu repositorio.
  6. Realiza el primer commit y el primer push.
  7. Abre la documentación con MkDocs.
